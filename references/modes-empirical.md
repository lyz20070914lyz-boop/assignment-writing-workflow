# Research Report 与 Proposal

## 范例选择

比较课程范例和同领域可靠范例。课程模板、rubric 和实际研究设计高于发表论文或 protocol 的结构与密度。

## Research Report

使用研究问题、假设或目的陈述统领全文。检查变量、方法、结果和结论是否回答同一问题。

范例重点：必需结构、变量定义、方法可复核程度、结果呈现、结果与解释边界、限制和意义。

写作逻辑：只报告用户提供或可核实的方法、分析和结果。不得补全数据、步骤、统计量或显著性。按课程规则区分 results 与 discussion；discussion 解释结果、比较证据、处理替代解释和范围，不重复结果列表。

默认顺序：method → results → discussion → introduction → abstract。没有已核实的分析输出时，不写 results。

## Proposal

使用研究缺口、研究问题和拟议贡献统领全文。缺口必须有证据；问题、设计、样本、测量、流程和分析必须对应。

范例重点：问题与缺口、问题或假设、方法对应关系、可行性、分析计划、伦理、限制和预期贡献。

改善 methodology 时逐项检查：研究问题对应、设计选择理由、样本与招募、变量或材料、程序、分析计划、质量控制、伦理、可行性和限制。区分“课程要求缺失”“信息未提供”和“设计问题”，不补造研究细节。

默认顺序：研究问题与范围 → 方法和可行性 → 分析计划 → 问题与缺口证据 → rationale → 伦理、限制和贡献 → 必需摘要。预测不得写成已观察结果。
