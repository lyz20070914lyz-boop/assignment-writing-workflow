# Research Report 与 Proposal

## 共同范例规则

同时比较课程范例和同领域可靠范例。发表论文或 protocol 的篇幅、格式和报告密度不能覆盖课程模板。

## Research Report

范例重点：必需结构、变量定义、方法可复核程度、结果呈现、结果与解释的边界、假设、限制和意义。

写作逻辑：只报告真实方法、分析和结果。不得补全缺失数据或步骤。按课程规则区分结果与讨论。

默认顺序：method → results → discussion → introduction → abstract。没有已核实的分析输出时，不写 results。

## Proposal

范例重点：问题与研究缺口、研究问题或假设、方法对应关系、可行性、分析计划、伦理、限制和预期贡献。

写作逻辑：先确定有证据支持的问题和缺口，再提出研究问题。设计、样本、测量、流程和分析必须回答该问题。预测不能写成已经观察到的结果。

默认顺序：研究问题与范围 → 方法和可行性 → 分析计划 → 问题与缺口证据 → rationale → 伦理、限制和贡献 → 必需摘要。
