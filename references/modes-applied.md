# Reflection 与 Case Study

## 共同范例规则

同时比较教师范例和大学、专业机构或学科内可靠范例。不得复制他人的经历、个人声音或案例事实。

## Reflection

先确认课程要求的 reflective model 和第一人称规则。

范例重点：描述与分析比例、经历与理论连接、立场和假设、学习证据、后续行动。

写作逻辑：只使用用户提供的经历。按经历或决定 → 解释 → 理论或证据 → 评价 → 学习 → 后续行动推进。不得编造感受、动机、文化背景或个人信息。

默认顺序：选择经历 → 确定分析问题 → 连接理论和证据 → 评价学习 → 后续行动 → 开头和结尾。

## Case Study

先确认任务要求 diagnosis、formulation、ethical analysis、decision analysis 或 recommendation 中的哪一种。

范例重点：案例事实选择、事实与推断边界、框架应用、替代解释、建议、风险、伦理和不确定性。

写作逻辑：把案例事实与解释分开。每个判断和建议必须连接案例证据与外部证据。说明替代解释和不确定性。不得补造案例事实。

默认顺序：相关事实 → 问题 → 框架分析 → 替代解释与不确定性 → 建议 → 开头和结尾。
