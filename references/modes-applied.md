# Reflection 与 Case Study

## 范例选择

比较教师范例和大学、专业机构或学科内可靠范例。课程要求高于外部范例。不得复制他人的经历、声音或案例事实。

## Reflection

先确认 reflective model、第一人称规则和评价重点。使用反思主线统领经历、理论、分析、学习结果和后续行动，不强套 argumentative thesis。

范例重点：描述与分析比例、经历与理论连接、假设与立场、学习证据和行动的具体性。

写作逻辑：只使用用户提供的经历。按经历或决定 → 解释 → 理论或证据 → 评价 → 学习 → 后续行动推进。区分发生了什么、为什么重要、理论如何改变理解以及未来如何行动。

默认顺序：选择经历 → 确定分析问题 → 连接理论和证据 → 评价学习 → 后续行动 → 开头和结尾。不得编造感受、动机、文化背景或个人信息。

## Case Study

先确认任务要求 diagnosis、formulation、ethical analysis、decision analysis 或 recommendation 中的哪一种。使用核心问题、判断标准和建议方向统领全文。

范例重点：案例事实选择、事实与推断边界、框架应用、替代解释、建议、风险、伦理和不确定性。

写作逻辑：分开记录案例事实、解释和外部证据。每个判断说明证据、推理、替代解释和不确定性；每项建议说明适用条件、风险和与核心问题的关系。

默认顺序：相关事实 → 核心问题 → 框架分析 → 替代解释与不确定性 → 建议 → 开头和结尾。不得补造案例事实。
