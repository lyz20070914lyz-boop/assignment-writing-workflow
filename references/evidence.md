# 文献、支持强度与 APA 7

## 证据需求与搜索

先把中心控制项和每个主要主张转成证据需求：概念、机制、群体、情境、方法、比较、分歧和限制。经验性主张优先匹配的原始研究；领域综合使用合适类型的 review。

按以下顺序核验身份：出版商或 DOI 注册记录 → 学术数据库或权威索引 → Google Scholar 可查询性。Google Scholar 不能单独证明来源身份或 DOI 正确。

检索不可用时，记录限制并标记 `需要人工确认`；不得把未核实项目当作已验证来源。

## 身份与内容状态

每项记录作者、年份、完整标题、出版来源、卷期页或文章号、DOI、稳定链接和核查来源。

- 有 DOI：解析 DOI，并核对作者、标题、年份和出版来源；关键字段不符时不得使用该 DOI。
- 无 DOI：用出版商、期刊、图书馆、repository 或权威索引确认来源存在；审计记录 `no DOI found`，不得补造。

分别记录来源身份和可用内容：

- `全文已核对`
- `摘要已核对`
- `仅元数据核对`
- `需要人工确认`

仅元数据不能支持结果性主张。摘要只能支持其中明确陈述的内容，不能扩展为全文结论。

## 相关性与支持强度

对每个需要引用的句子或具体主张记录：

| 字段 | 内容 |
|---|---|
| 句子或主张 | 正文实际表达 |
| 主张类型 | 描述、相关、预测、机制、因果、评价或建议 |
| 来源与定位 | 作者年份及页码/section（仅在已核实时） |
| 内容状态 | 全文、摘要、元数据或待确认 |
| 支持强度 | 直接、条件性、间接、仅背景或不支持 |
| 边界 | 样本、方法、群体、情境、时间和作者限定 |
| 处理 | 保留、缩小主张、补来源、改引原始研究或删除 |

`直接` 表示来源在匹配范围内明确支持该主张；`条件性` 表示需保留限定；`间接` 只能支持推理的一部分；`仅背景` 不得作为核心结论证据；`不支持` 必须更换来源或修改正文。

重点拦截：

- 来源真实但不支持句中说法。
- 把相关、预测或关联写成因果或机制。
- 把作者推测、讨论或未来方向写成研究发现。
- 把 review 当作原始研究。
- 从摘要推断未报告的方法、结果或限制。
- 忽略样本、测量、方法、群体或情境边界。
- 使用未核实的直接引文、页码或 DOI。

## 筛选

按题目匹配、论证作用、研究设计、来源质量、群体和情境、时效、rubric 作用、重复程度和核查状态筛选。每篇入选来源写明支持的具体主张、支持强度和限制；每篇淘汰来源保留一个原因。

不得只称某来源“最好”。必须给出比较标准。存在分歧时呈现分歧；需要原始研究且可获得时，引用原始研究。

## 引用与 APA 7 审计

从最新版正文提取全部 in-text citations，逐项完成双向核对：

1. 每个正文引用都对应一条身份已解决的 Reference。
2. 每条 Reference 都在正文实际出现。
3. 作者、年份、标题、出版信息和 DOI 互相对应。
4. 引用位置与句中主张相符，避免 citation dumping。
5. 直接引文的文字、页码或段落号已经核实。
6. APA 7 排序、作者、年份、标题、来源和 DOI 格式正确。

输出分开：

1. `APA 7 References list`：只放可用条目，不附 `no DOI found`。
2. `需要人工确认`：身份、定位或引用关系未解决的项目。
3. `DOI核对清单`：DOI 状态、核查来源和正文出现状态；无 DOI 在此标记。
